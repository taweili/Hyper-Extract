## 1. 概述

### 1.1 什么是知识模板

知识模板是 Hyper-Extract 系统中用于定义结构化信息提取规则的配置文件。通过声明式的 YAML 格式，开发者可以定义：

- **数据结构**：提取结果的输出定义
- **提取逻辑**：LLM 应该从文本中提取什么
- **标识规则**：如何唯一识别实体和关系
- **展示方式**：结果如何呈现给用户

### 1.2 YAML 格式简介

YAML（YAML Ain't Markup Language）是一种人类可读的数据序列化格式，广泛用于配置文件。

**参考文档**：[YAML 官方文档](https://yaml.org/)

**基本特点**：

- 使用缩进表示层级关系
- 键值对使用 `key: value` 格式
- 数组使用 `-` 开头
- 多行文本使用 `|` 或 `>` 符号

### 1.3 模板系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      YAML 配置文件                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────────────┐  │
│  │language │ │  name   │ │  type   │ │ tags │  │
│  │   ...   │ │         │ │         │ │   identifiers   │  │
│  │         │ │         │ │         │ │    display      │  │
│  └─────────┘ └─────────┘ └─────────┘ └─────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                   TemplateFactory                            │
│         (将 YAML 配置转换为运行时模板)                       │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                   提取结果 (结构化数据)                       │
│              Model / List / Set / Graph                      │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 语言配置说明

**重要**：YAML 配置中的 `language` 字段用于指定默认语言。当设置为单语言（如 `zh`）时，描述字段可以直接书写，无需额外的语言键。

```yaml
# language: zh 时（单语言）
# 描述字段直接书写，不需要额外的 zh: 键
language: zh

name: EarningsCallSummary
type: model
tags: [finance]
description: '财报电话会议摘要 - 从财报电话会议记录中提取关键财务指标'

output:
  description: '财报电话会议的结构化摘要'
  fields:
    - name: company_name
      type: string
      description: '公司名称'
```

**多语言情况**：当需要支持多语言时，才需要使用语言键格式：

```yaml
language: zh

description:
  zh: "中文描述"
  en: "English description"
```

### 1.5 支持的 AutoType 类型

| 类型                      | YAML 值                  | 说明        |
| ----------------------- | ----------------------- | --------- |
| AutoModel               | `model`                 | 提取单个结构化对象 |
| AutoList                | `list`                  | 提取对象列表    |
| AutoSet                 | `set`                   | 提取去重实体集合  |
| AutoGraph               | `graph`                 | 简单二元关系图谱  |
| AutoHypergraph          | `hypergraph`            | 复杂多元关系图谱  |
| AutoTemporalGraph       | `temporal_graph`        | 时序图谱      |
| AutoSpatialGraph        | `spatial_graph`         | 空间图谱      |
| AutoSpatioTemporalGraph | `spatio_temporal_graph` | 时空图谱      |

## 2. 快速入门

### 2.1 最简配置 (AutoModel)

```yaml
language: zh

name: MyTemplate
type: model
tags: [general]
description: '模板简短描述'

output:
  description: '输出结构描述'
  fields:
    - name: field_name
      type: string
      description: '字段描述'

guideline:
  target: '你是一位专家，请从文本中提取信息。'
  rules:
    - '提取规则1'
    - '提取规则2'

display:
  label: 'field_name'
```

### 2.2 完整配置示例 (AutoGraph)

```yaml
language: zh

name: MyKnowledgeGraph
type: graph
tags: [domain_name]
description: '知识图谱描述'

output:
  description: '知识图谱的数据结构定义'
  entities:
    description: '实体定义'
    fields:
      - name: name
        type: string
        description: '实体名称'
      - name: category
        type: string
        description: '实体类别'
  relations:
    description: '关系定义'
    fields:
      - name: source
        type: string
        description: '源实体'
      - name: target
        type: string
        description: '目标实体'
      - name: relationType
        type: string
        description: '关系类型'

guideline:
  target: '你是一位领域专家，请从文本中提取知识图谱关系。'
  rules_for_entities:
    - '实体提取规则'
  rules_for_relations:
    - '关系提取规则'

identifiers:
  entity_id: 'name'
  relation_id: '{source}|{relationType}|{target}'
  relation_members:
    source: 'source'
    target: 'target'

options:
  extraction_mode: two_stage

display:
  entity_label: '{name} ({category})'
  relation_label: '{relationType}'
```

## 3. 配置字段详解

### 3.1 通用必需字段

以下字段是所有模板都必须配置的：

| 字段            | 说明        | 示例                             |
| ------------- | --------- | ------------------------------ |
| `language`    | 语言代码      | `zh`                           |
| `name`        | 模板类名，唯一标识 | `EarningsCallSummary`          |
| `type`        | 模板类型      | `model`, `graph`, `list` 等     |
| `tags`        | 领域标签数组    | `[finance]`, `[medicine, tcm]` |
| `description` | 模板描述      | 简要说明模板用途                       |

### 3.2 其他字段说明

以下字段的具体配置请参考各 AutoType 类型章节：

| 字段                         | 说明     |
| -------------------------- | ------ |
| `output`                   | 输出定义   |
| `output.entities`          | 实体定义   |
| `output.relations`         | 关系定义   |
| `guideline`                | 提取指南   |
| `identifiers`              | 标识配置   |
| `options`                  | 运行时参数  |
| `display`                  | 展示配置   |

### 3.3 Output 配置说明

`output` 字段用于定义输出数据结构，根据 AutoType 类型有不同的配置方式：

#### 3.3.1 非图类型（model/list/set）

```yaml
output:
  description: '输出结构描述'
  fields:
    - name: field_name
      type: string
      description: '字段描述'
      required: true
      default: ''
```

#### 3.3.2 图类型（graph/hypergraph等）

```yaml
output:
  description: '输出结构描述'
  entities:
    description: '实体定义描述'
    fields:
      - name: name
        type: string
        description: '实体名称'
  relations:
    description: '关系定义描述'
    fields:
      - name: source
        type: string
        description: '源实体'
```

### 3.4 Options 配置说明

`options` 字段用于配置运行时参数，支持以下配置项：

#### 3.4.1 Merge Strategy 配置

合并策略用于定义当检测到重复实体时如何处理。有以下预定义策略可供选择：

| 策略                   | YAML值                 | 说明             |
| -------------------- | --------------------- | -------------- |
| MERGE\_FIELD         | `merge_field`         | 非空值覆盖，列表追加（默认） |
| KEEP\_INCOMING       | `keep_incoming`       | 最新数据胜出         |
| KEEP\_EXISTING       | `keep_existing`       | 保留首次数据         |
| LLM.BALANCED         | `llm_balanced`        | LLM智能平衡（默认）    |
| LLM.PREFER\_INCOMING | `llm_prefer_incoming` | 偏向新数据          |
| LLM.PREFER\_EXISTING | `llm_prefer_existing` | 偏向旧数据          |

**不同 AutoType 的策略参数名**：

| AutoType                 | 参数名                                           | 示例                                                                  |
| ------------------------ | --------------------------------------------- | ------------------------------------------------------------------- |
| AutoModel                | `merge_strategy`                              | `merge_strategy: llm_balanced`                                      |
| AutoList                 | -                                             | 无（列表类型不需要合并）                                                        |
| AutoSet                  | `merge_strategy`                              | `merge_strategy: keep_incoming`                                     |
| AutoGraph/AutoHypergraph | `entity_merge_strategy` + `relation_merge_strategy` | `entity_merge_strategy: llm_balanced`<br>`relation_merge_strategy: merge_field` |
| AutoTemporalGraph        | `entity_merge_strategy` + `relation_merge_strategy` | 同上                                                                  |
| AutoSpatialGraph         | `entity_merge_strategy` + `relation_merge_strategy` | 同上                                                                  |
| AutoSpatioTemporalGraph  | `entity_merge_strategy` + `relation_merge_strategy` | 同上                                                                  |

**示例配置**：

```yaml
# Model/Set 类型
options:
  merge_strategy: llm_balanced

# Graph 类型
options:
  entity_merge_strategy: llm_balanced
  relation_merge_strategy: merge_field
```

#### 3.4.2 其他 Options 配置

| 参数                | 类型   | 默认值         | 说明                             |
| ----------------- | ---- | ----------- | ------------------------------ |
| `extraction_mode` | str  | `two_stage` | 提取模式：`one_stage` 或 `two_stage` |
| `chunk_size`      | int  | `2048`      | 文本分块大小                         |
| `chunk_overlap`   | int  | `256`       | 分块重叠大小                         |
| `max_workers`     | int  | `10`        | 最大工作线程数                        |
| `verbose`         | bool | `false`     | 是否输出详细信息                       |

**时序图专用参数**：
\| `observation_time` | str | - | 观察时间点 |

**空间图专用参数**：
\| `observation_location` | str | - | 观察位置 |

### 3.5 Rules 字段格式说明

`guideline` 中的 `rules`、`rules_for_entities`、`rules_for_relations`、`rules_for_time`、`rules_for_location` 等字段使用 List 格式：

```yaml
rules:
  - '提取规则1'
  - '提取规则2'
```

解析时会自动转换为有序编号字符串：

```
1. 提取规则1
2. 提取规则2
```

**多语言情况**：

```yaml
rules:
  zh:
    - '中文规则1'
    - '中文规则2'
  en:
    - 'English rule 1'
    - 'English rule 2'
```

## 4. 多语言配置指南

### 4.1 单语言格式

当 `language` 设置为单一语言（如 `zh`）时，描述字段直接书写：

```yaml
language: zh
name: EarningsCallSummary
type: model
tags: [finance]

description: '财报电话会议摘要 - 从财报电话会议记录中提取关键财务指标'

  description: '财报电话会议的结构化摘要'
  fields:
    - name: company_name
      type: string
      description: '公司名称'
```

### 4.2 多语言格式

当需要支持多语言时，使用语言键格式：

```yaml
language: zh

name: EarningsCallSummary
type: model
tags: [finance]
description:
  zh: '中文描述'
  en: 'English description'

output:
  description:
    zh: '财报电话会议的结构化摘要'
    en: 'Structured summary of earnings call'
  fields:
    - name: company_name
      type: string
      description:
        zh: '公司名称'
        en: 'Company name'  
```

## 5. AutoType 类型指南

### 5.1 AutoType 类型概览表

| 类型                      | 值                       | Output                      | Identifiers                       | Guideline                                                                     | Display                   |
| ----------------------- | ----------------------- | --------------------------- | --------------------------------- | ------------------------------------------------------------------------------------ | ------------------------- |
| AutoModel               | `model`                 | `output`                    | -                                 | target, rules                                                                        | label                     |
| AutoList                | `list`                  | `output`                    | -                                 | target, rules                                                                        | label                     |
| AutoSet                 | `set`                   | `output`                    | item\_id                          | target, rules                                                                        | label                     |
| AutoGraph               | `graph`                 | entities + relations | entity\_id, relation\_id, relation\_members | target, rules\_for\_entities, rules\_for\_relations                                         | entity\_label + relation\_label |
| AutoHypergraph          | `hypergraph`            | entities + relations | entity\_id, relation\_id, relation\_members | target, rules\_for\_entities, rules\_for\_relations                                         | entity\_label + relation\_label |
| AutoTemporalGraph       | `temporal_graph`        | entities + relations | + time\_field                     | target, rules\_for\_entities, rules\_for\_relations, rules\_for\_time                       | entity\_label + relation\_label |
| AutoSpatialGraph        | `spatial_graph`         | entities + relations | + location\_field                 | target, rules\_for\_entities, rules\_for\_relations, rules\_for\_location                   | entity\_label + relation\_label |
| AutoSpatioTemporalGraph | `spatio_temporal_graph` | entities + relations | + time\_field + location\_field   | target, rules\_for\_entities, rules\_for\_relations, rules\_for\_time, rules\_for\_location | entity\_label + relation\_label |

### 5.2 AutoModel - 结构化模型

用于提取单个结构化对象，包含多个字段。

**output 定义**：

```yaml
output:
  description: '输出结构描述'
  fields:
    - name: company_name
      type: string
      description: '公司名称'
      required: true
      default: ''
    - name: revenue
      type: string
      description: '营收金额'
      required: false
      default: 'N/A'
```

**guideline 编写**：

```yaml
guideline:
  target: '你是一位财务分析师，请从财报中提取关键信息。'
  rules: 
    - '每个字段对应一个独立的信息项'
    - '数据与原文保持一致'
```

**display 配置**：

```yaml
display:
  label: '{company_name}'
```

**完整示例**：

```yaml
language: zh

name: EarningsCallSummary
type: model
tags: [finance]
description: '财报电话会议摘要 - 从财报电话会议记录中提取关键财务指标'

output:
  description: '财报电话会议的结构化摘要'
  fields:
    - name: company_name
      type: string
      description: '公司名称'
    - name: revenue
      type: string
      description: '营收金额'

guideline:
  target: '你是一位财务分析师，请从财报中提取关键信息。'
  rules:
    - '每个字段对应一个独立的信息项'
    - '数据与原文保持一致'

display:
  label: '{company_name}'
```

### 5.3 AutoList - 列表

用于提取对象列表，每个对象结构相同。

**output 定义**：

```yaml
output:
  description: '关键词列表'
  fields:
    - name: term
      type: string
      description: '关键词'
    - name: frequency
      type: int
      description: '出现频次'
```

**display 配置**：

```yaml
display:
  label: term
```

**完整示例**：

```yaml
language: zh

name: KeywordList
type: list
tags: [general]
description: '关键词列表 - 从文本中提取所有关键词'

output:
  description: '关键词列表'
  fields:
    - name: term
      type: string
      description: '关键词'

guideline:
  target: '你是一位关键词提取专家'
  rules:
    - '提取文本中的所有关键词'

display:
  label: term
```

### 5.4 AutoSet - 集合

用于提取去重实体集合。

**output 定义**：同 AutoList

**identifiers 配置**：

```yaml
identifiers:
  item_id: name
```

**完整示例**：

```yaml
language: zh

name: EntityRegistry
type: set
tags: [general]
description: '实体注册表 - 从文本中提取所有实体并去重'

output:
  description: '实体注册表'
  fields:
    - name: name
      type: string
      description: '实体名称'
    - name: category
      type: string
      description: '实体类型'

guideline:
  target: '你是一位实体识别专家'
  rules:
    - '提取所有实体并去重'

identifiers:
  item_id: name

display:
  label: name
```

### 5.5 AutoGraph - 简单二元关系图谱

用于提取实体和它们之间的二元关系。

**output.entities 定义**：

```yaml
output:
  entities:
    description: '实体定义'
    fields:
      - name: name
        type: string
        description: '实体名称'
      - name: category
        type: string
        description: '实体类别'
```

**output.relations 定义**：

```yaml
output:
  relations:
    description: '关系定义'
    fields:
      - name: source
        type: string
        description: '源实体'
      - name: target
        type: string
        description: '目标实体'
      - name: relationType
        type: string
        description: '关系类型'
```

**identifiers 配置**：

```yaml
identifiers:
  entity_id: name
  relation_id: '{source}|{relationType}|{target}'
  relation_members:
    source: source
    target: target
```

**guideline 编写**：

```yaml
guideline:
  target: '你是一位知识图谱专家，请从文本中提取实体关系。'
  rules_for_entities:
    - '实体名称与原文保持一致'
    - '每个实体必须有明确的类别'
  rules_for_relations:
    - 'source 和 target 都必须是已知节点'
    - 'relationType 必须是预定义的关系类型'
```

**display 配置**：

```yaml
display:
  entity_label: '{name} ({category})'
  relation_label: '{relationType}'
```

### 5.6 AutoHypergraph - 复杂多元关系图谱

用于建模多个实体之间的复杂关联关系。

**与 AutoGraph 的区别**：

- AutoGraph 是**二元关系**：每条边连接两个节点
- AutoHypergraph 是**多元关系**：一条边可以连接任意数量的节点

**output.relations 定义**：

```yaml
output:
  relations:
    description: '关系定义'
    fields:
      - name: mechanism_name
        type: string
        description: '机制名称'
      - name: participating_entities
        type: array
        description: '参与的实体列表'
      - name: relationType
        type: string
        description: '关系类型'
      - name: outcome
        type: string
        description: '结果'
```

**identifiers 配置**：

```yaml
identifiers:
  entity_id: 'name'
  relation_id: 'mechanism_name'
  relation_members:
    source: 'participating_entities'
    target: 'outcome'
```

### 5.7 AutoTemporalGraph - 时序图谱

用于提取带有时间信息的实体关系。

**特有 identifiers 配置**：

```yaml
identifiers:
  entity_id: 'name'
  relation_id: '{source}|{relationType}|{target}'
  relation_members:
    source: 'source'
    target: 'target'
  time_field: 'fiscal_period'
```

**特有 guideline 编写**：

```yaml
guideline:
  rules_for_time:
    - '当前观察日期：{observation_time}'
    - "'去年' → {observation_time} 的前一年"
```

**特有 options 配置**：

```yaml
options:
  extraction_mode: 'two_stage'
  observation_time: '2024-01-01'
```

### 5.8 AutoSpatialGraph - 空间图谱

用于提取带有位置信息的实体关系。

**特有 identifiers 配置**：

```yaml
identifiers:
  location_field: 'location'
```

**特有 guideline 编写**：

```yaml
guideline:
  rules_for_location:
    - '当前观察位置：{observation_location}'
    - "'这里' → {observation_location}"
```

**特有 options 配置**：

```yaml
options:
  observation_location: '北京'
```

### 5.9 AutoSpatioTemporalGraph - 时空图谱

综合 AutoTemporalGraph 和 AutoSpatialGraph 的所有配置。

**同时包含**：

- `time_field` + `rules_for_time` + `observation_time`
- `location_field` + `rules_for_location` + `observation_location`

## 6. 最佳实践

### 6.1 命名规范

- **name**: 使用 PascalCase，如 `EarningsCallSummary`
- **tags**: 使用小写，如 `[finance]`, `[medicine]`
- **字段名**: 使用 snake\_case，如 `company_name`, `fiscal_period`
- **关系类型**: 使用中文，如 `拥有`、`影响`

### 6.2 description 撰写

- 简洁明了，一句话说明模板用途
- 说明输入文本来源
- 说明提取目标

**示例**：

```yaml
description: '财报电话会议摘要 - 从财报电话会议记录中提取关键财务指标'
```

### 6.3 测试验证

创建模板后，建议：

1. 使用真实文本测试提取效果
2. 检查字段覆盖是否完整
3. 验证边节点关系是否正确
4. 确认时间/位置解析是否准确
