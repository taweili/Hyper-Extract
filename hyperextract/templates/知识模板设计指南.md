## 1. 概述

### 1.1 什么是知识模板

知识模板是 Hyper-Extract 系统中用于定义结构化信息提取规则的配置文件。通过声明式的 YAML 格式，开发者可以定义：

- **数据结构**：提取结果的 Schema 定义
- **提取逻辑**：LLM 应该从文本中提取什么
- **标识规则**：如何唯一识别实体和关系
- **展示方式**：结果如何呈现给用户

### 1.2 YAML 格式简介

YAML（YAML Ain't Markup Language）是一种人类可读的数据序列化格式，广泛用于配置文件。

**参考文档**：[YAML 官方文档](https://yaml.org/)

**基本特点**：
- 使用缩进表示层级关系
- 键值对使用 `key: value` 格式
- 数组使用 `-` 开头
- 多行文本使用 `|` 或 `>` 符号

### 1.3 模板系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      YAML 配置文件                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────────────┐  │
│  │  name   │ │ autotype│ │  schema │ │ extraction_guide│  │
│  │   tag   │ │ language│ │   ...   │ │   identifiers   │  │
│  │   ...   │ │         │ │         │ │    display      │  │
│  └─────────┘ └─────────┘ └─────────┘ └─────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                   TemplateFactory                            │
│         (将 YAML 配置转换为运行时模板)                       │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                   提取结果 (结构化数据)                       │
│              Model / List / Set / Graph                      │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 语言配置说明

**重要**：YAML 配置中的 `language` 字段用于指定默认语言。当设置为单语言（如 `zh`）时，描述字段可以直接书写，无需额外的语言键。

```yaml
# language: zh 时（单语言）
# 描述字段直接书写，不需要额外的 zh: 键
name: EarningsCallSummary
language: zh

description: 财报电话会议摘要 - 从财报电话会议记录中提取关键财务指标

schema:
  fields:
    - name: company_name
      type: string
      description: 公司名称
```

**多语言情况**：当需要支持多语言时，才需要使用语言键格式：

```yaml
language: zh

description:
  zh: 中文描述
  en: English description
```

### 1.5 支持的 AutoType 类型

| 类型 | YAML 值 | 说明 |
|------|---------|------|
| AutoModel | `model` | 提取单个结构化对象 |
| AutoList | `list` | 提取对象列表 |
| AutoSet | `set` | 提取去重实体集合 |
| AutoGraph | `graph` | 简单二元关系图谱 |
| AutoHypergraph | `hypergraph` | 复杂多元关系图谱 |
| AutoTemporalGraph | `temporal_graph` | 时序图谱 |
| AutoSpatialGraph | `spatial_graph` | 空间图谱 |
| AutoSpatioTemporalGraph | `spatio_temporal_graph` | 时空图谱 |

## 2. 快速入门

### 2.1 最简配置 (AutoModel)

```yaml
# 最简单的 model 类型配置
name: MyTemplate
autotype: model
tag: [general]

language: zh

description: 模板简短描述

schema:
  fields:
    - name: field_name
      type: string
      description: 字段描述

extraction_guide:
  target: 你是一位专家，请从文本中提取信息。

  rules: |
    1. 提取规则1
    2. 提取规则2

identifiers: {}

parameters: {}

display:
  label: field_name
```

### 2.2 完整配置示例 (AutoGraph)

```yaml
name: MyKnowledgeGraph
autotype: graph
tag: [domain_name]

language: zh

description: 知识图谱描述

node_schema:
  fields:
    - name: name
      type: string
      description: 实体名称
    - name: category
      type: string
      description: 实体类别

edge_schema:
  fields:
    - name: source
      type: string
      description: 源实体
    - name: target
      type: string
      description: 目标实体
    - name: relationType
      type: string
      description: 关系类型

extraction_guide:
  target: 你是一位领域专家...

  rules: |
    1. 规则1
    2. 规则2

  rules_for_nodes: |
    1. 节点提取规则

  rules_for_edges: |
    1. 边提取规则

identifiers:
  node_id: name
  edge_id: "{source}|{relationType}|{target}"
  edge_members:
    source: source
    target: target

parameters:
  extraction_mode: two_stage

display:
  node_label: "{name} ({category})"
  edge_label: "{relationType}"
```

## 3. 配置字段详解

### 3.1 通用必需字段

以下字段是所有模板都必须配置的：

| 字段 | 说明 | 示例 |
|------|------|------|
| `name` | 模板类名，唯一标识 | `EarningsCallSummary` |
| `autotype` | 模板类型 | `model`, `graph`, `list` 等 |
| `tag` | 领域标签数组 | `[finance]`, `[medicine, tcm]` |
| `language` | 语言代码 | `zh` |
| `description` | 模板描述 | 简要说明模板用途 |

### 3.2 其他字段说明

以下字段的具体配置请参考各 AutoType 类型章节：

| 字段 | 说明 |
|------|------|
| `schema` / `item_schema` / `node_schema` / `edge_schema` | 数据结构定义 |
| `extraction_guide` | 提取指南 |
| `identifiers` | 标识配置 |
| `parameters` | 运行时参数 |
| `display` | 展示配置 |

## 4. 多语言配置指南

### 4.1 单语言格式

当 `language` 设置为单一语言（如 `zh`）时，描述字段直接书写：

```yaml
name: EarningsCallSummary
language: zh

description: 财报电话会议摘要 - 从财报电话会议记录中提取关键财务指标

  fields:
    - name: company_name
      type: string
      description: 公司名称
```

### 4.2 多语言格式

当需要支持多语言时，使用语言键格式：

```yaml
language: zh

description:
  zh: 中文描述
  en: English description

schema:
  fields:
    - name: company_name
      type: string
      description:
        zh: 公司名称
        en: Company name
```

## 5. AutoType 类型指南

### 5.1 AutoType 类型概览表

| 类型 | 值 | Schema | Identifiers | Extraction Guide | Display |
|------|------|--------|-------------|-------------------|---------|
| AutoModel | `model` | `schema` | - | target, rules | label |
| AutoList | `list` | `item_schema` | - | target, rules | label |
| AutoSet | `set` | `item_schema` | item_id | target, rules | label |
| AutoGraph | `graph` | node_schema + edge_schema | node_id, edge_id, edge_members | target, rules, rules_for_nodes, rules_for_edges | node_label + edge_label |
| AutoHypergraph | `hypergraph` | node_schema + edge_schema | node_id, edge_id, edge_members | target, rules, rules_for_nodes, rules_for_edges | node_label + edge_label |
| AutoTemporalGraph | `temporal_graph` | node_schema + edge_schema | + time_field | + rules_for_time | node_label + edge_label |
| AutoSpatialGraph | `spatial_graph` | node_schema + edge_schema | + location_field | + rules_for_location | node_label + edge_label |
| AutoSpatioTemporalGraph | `spatio_temporal_graph` | node_schema + edge_schema | + time_field + location_field | + rules_for_time + rules_for_location | node_label + edge_label |

### 5.2 AutoModel - 结构化模型

用于提取单个结构化对象，包含多个字段。

**schema 定义**：
```yaml
schema:
  fields:
    - name: company_name
      type: string
      description: 公司名称
      required: true
      default: ""
    - name: revenue
      type: string
      description: 营收金额
      required: false
      default: "N/A"
```

**extraction_guide 编写**：
```yaml
extraction_guide:
  target: 你是一位财务分析师，请从财报中提取关键信息。

  rules: |
    1. 每个字段对应一个独立的信息项
    2. 数据与原文保持一致
```

**display 配置**：
```yaml
display:
  label: "{company_name}"
```

**完整示例**：
```yaml
name: EarningsCallSummary
autotype: model
tag: [finance]

language: zh

description: 财报电话会议摘要

schema:
  fields:
    - name: company_name
      type: string
      description: 公司名称
    - name: revenue
      type: string
      description: 营收金额

extraction_guide:
  target: 你是一位财务分析师，请从财报中提取关键信息。

  rules: |
    1. 每个字段对应一个独立的信息项

identifiers: {}

display:
  label: "{company_name}"
```

### 5.3 AutoList - 列表

用于提取对象列表，每个对象结构相同。

**item_schema 定义**：
```yaml
item_schema:
  fields:
    - name: term
      type: string
      description: 关键词
    - name: frequency
      type: int
      description: 出现频次
```

**display 配置**：
```yaml
display:
  label: term
```

**完整示例**：
```yaml
name: KeywordList
autotype: list

language: zh

description: 关键词列表

item_schema:
  fields:
    - name: term
      type: string
      description: 关键词

extraction_guide:
  target: 你是一位关键词提取专家

  rules: |
    1. 提取文本中的所有关键词

identifiers: {}

display:
  label: term
```

### 5.4 AutoSet - 集合

用于提取去重实体集合。

**item_schema 定义**：同 AutoList

**identifiers 配置**：
```yaml
identifiers:
  item_id: name
```

**完整示例**：
```yaml
name: EntityRegistry
autotype: set

language: zh

description: 实体注册表

item_schema:
  fields:
    - name: name
      type: string
      description: 实体名称
    - name: category
      type: string
      description: 实体类型

extraction_guide:
  target: 你是一位实体识别专家

  rules: |
    1. 提取所有实体并去重

identifiers:
  item_id: name

display:
  label: name
```

### 5.5 AutoGraph - 简单二元关系图谱

用于提取实体和它们之间的二元关系。

**node_schema 定义**：
```yaml
node_schema:
  fields:
    - name: name
      type: string
      description: 实体名称
    - name: category
      type: string
      description: 实体类别
```

**edge_schema 定义**：
```yaml
edge_schema:
  fields:
    - name: source
      type: string
      description: 源实体
    - name: target
      type: string
      description: 目标实体
    - name: relationType
      type: string
      description: 关系类型
```

**identifiers 配置**：
```yaml
identifiers:
  node_id: name
  edge_id: "{source}|{relationType}|{target}"
  edge_members:
    source: source
    target: target
```

**extraction_guide 编写**：
```yaml
extraction_guide:
  target: 你是一位知识图谱专家...

  rules: |
    1. 每个节点对应一个独立实体

  rules_for_nodes: |
    1. 实体名称与原文保持一致

  rules_for_edges: |
    1. source 和 target 都必须是已知节点
```

**display 配置**：
```yaml
display:
  node_label: "{name} ({category})"
  edge_label: "{relationType}"
```

### 5.6 AutoHypergraph - 复杂多元关系图谱

用于建模多个实体之间的复杂关联关系。

**与 AutoGraph 的区别**：
- AutoGraph 是**二元关系**：每条边连接两个节点
- AutoHypergraph 是**多元关系**：一条边可以连接任意数量的节点

**edge_schema 定义**：
```yaml
edge_schema:
  fields:
    - name: mechanism_name
      type: string
      description: 机制名称
    - name: participating_entities
      type: array
      description: 参与的实体列表
    - name: relationType
      type: string
      description: 关系类型
    - name: outcome
      type: string
      description: 结果
```

**identifiers 配置**：
```yaml
identifiers:
  node_id: name
  edge_id: mechanism_name
  edge_members:
    source: participating_entities
    target: outcome
```

### 5.7 AutoTemporalGraph - 时序图谱

用于提取带有时间信息的实体关系。

**特有 identifiers 配置**：
```yaml
identifiers:
  node_id: name
  edge_id: "{source}|{relationType}|{target}"
  edge_members:
    source: source
    target: target
  time_field: fiscal_period
```

**特有 extraction_guide 编写**：
```yaml
extraction_guide:
  rules_for_time: |
    当前观察日期：{observation_time}
    1. "去年" → {observation_time} 的前一年
```

**特有 parameters 配置**：
```yaml
parameters:
  extraction_mode: two_stage
  observation_time: "2024-01-01"
```

### 5.8 AutoSpatialGraph - 空间图谱

用于提取带有位置信息的实体关系。

**特有 identifiers 配置**：
```yaml
identifiers:
  location_field: location
```

**特有 extraction_guide 编写**：
```yaml
extraction_guide:
  rules_for_location: |
    当前观察位置：{observation_location}
    1. "这里" → {observation_location}
```

**特有 parameters 配置**：
```yaml
parameters:
  observation_location: "北京"
```

### 5.9 AutoSpatioTemporalGraph - 时空图谱

综合 AutoTemporalGraph 和 AutoSpatialGraph 的所有配置。

**同时包含**：
- `time_field` + `rules_for_time` + `observation_time`
- `location_field` + `rules_for_location` + `observation_location`

## 6. 最佳实践

### 6.1 命名规范

- **name**: 使用 PascalCase，如 `EarningsCallSummary`
- **tag**: 使用小写，如 `[finance]`, `[medicine]`
- **字段名**: 使用 snake_case，如 `company_name`, `fiscal_period`
- **关系类型**: 使用中文，如 `拥有`、`影响`

### 6.2 description 撰写

- 简洁明了，一句话说明模板用途
- 说明输入文本来源
- 说明提取目标

**示例**：
```yaml
description: 财报电话会议摘要 - 从财报电话会议记录中提取关键财务指标
```

### 6.3 测试验证

创建模板后，建议：
1. 使用真实文本测试提取效果
2. 检查字段覆盖是否完整
3. 验证边节点关系是否正确
4. 确认时间/位置解析是否准确
